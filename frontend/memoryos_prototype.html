Same question — why did gradient descent need feature scaling?
Because your gradient descent implementation updates all weights using one shared learning rate — a concept from your Learning sessions on cost surfaces. Unscaled features create a stretched, elongated cost surface, so the shared step size overshoots on large-range features and crawls on small-range ones. Standardizing first makes the surface roughly circular, so one learning rate works evenly across all weights — which is exactly why your Optimized and Gradient Descent versions converge so much faster than Brute Force.
